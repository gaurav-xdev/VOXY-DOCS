# 022 - Command Schema Specification

## Purpose

Define the schema for all commands that can be issued to VOXY subsystems. Commands are distinct from events in that they require acknowledgment and may fail.

## Scope

- Command envelope schema
- Command payload definitions
- Acknowledgment and error response schemas
- Command routing rules
- Idempotency requirements

## Responsibilities

- Define canonical command schemas
- Enforce command validation
- Document command semantics
- Specify idempotency and retry behavior

## Non-Goals

- Command execution logic (subsystem responsibility)
- Command queue implementation
- Command scheduling

## Architecture Position

Command Schema defines the contract between command issuers (UI, plugins, external tools) and command executors (subsystems).

## Command Envelope Schema

```json
{
  "$schema": "http://json-schema.org/draft-07/schema#",
  "type": "object",
  "required": ["id", "command", "target", "payload", "issued_at"],
  "properties": {
    "id": {
      "type": "string",
      "format": "uuid"
    },
    "command": {
      "type": "string",
      "pattern": "^[a-z][a-z0-9_]*\.[a-z][a-z0-9_]*$",
      "description": "Command name (subsystem.action)"
    },
    "target": {
      "type": "string",
      "description": "Target subsystem or service"
    },
    "payload": {
      "type": "object"
    },
    "issued_at": {
      "type": "string",
      "format": "date-time"
    },
    "issuer": {
      "type": "string",
      "description": "Command issuer identity"
    },
    "idempotent": {
      "type": "boolean",
      "default": false
    },
    "timeout_ms": {
      "type": "integer",
      "default": 5000
    }
  }
}
```

## Command Registry

| Command | Target | Payload | Idempotent | Description |
|---------|--------|---------|------------|-------------|
| `audio.start_capture` | Audio Pipeline | { device_id } | Yes | Start audio capture |
| `audio.stop_capture` | Audio Pipeline | {} | Yes | Stop audio capture |
| `cognitive.process` | Cognitive Engine | { text, context } | No | Process user input |
| `cognitive.cancel` | Cognitive Engine | { request_id } | Yes | Cancel processing |
| `memory.query` | Memory Subsystem | { query, top_k } | Yes | Query memories |
| `memory.store` | Memory Subsystem | { entry } | No | Store memory |
| `automation.execute` | Automation Runtime | { action, target } | No | Execute UI action |
| `automation.capture` | Automation Runtime | { region } | Yes | Capture screen |
| `plugin.load` | Plugin Runtime | { plugin_id } | Yes | Load plugin |
| `plugin.unload` | Plugin Runtime | { plugin_id } | Yes | Unload plugin |
| `runtime.pause` | Runtime Manager | {} | Yes | Pause runtime |
| `runtime.resume` | Runtime Manager | {} | Yes | Resume runtime |
| `runtime.restart` | Runtime Manager | { subsystem } | Yes | Restart subsystem |

## Response Schema

```json
{
  "type": "object",
  "required": ["command_id", "status", "responded_at"],
  "properties": {
    "command_id": { "type": "string", "format": "uuid" },
    "status": { "type": "string", "enum": ["acknowledged", "completed", "failed", "timeout"] },
    "result": { "type": ["object", "null"] },
    "error": {
      "type": ["object", "null"],
      "properties": {
        "code": { "type": "string" },
        "message": { "type": "string" },
        "details": { "type": ["object", "null"] }
      }
    },
    "responded_at": { "type": "string", "format": "date-time" }
  }
}
```

## Idempotency

- Idempotent commands MAY be safely retried
- Non-idempotent commands require deduplication by command ID
- Command IDs are UUIDs generated by issuers
